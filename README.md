# VK advertising bot

Перенос Telegram-бота на VK API с сохранением основной бизнес-логики: рекламодатель, партнерские площадки, ручная модерация оплат, объявления, подписочные группы, рассылки, админка, статистика и планировщик публикаций.

## Быстрый запуск

1. Установить PostgreSQL и создать базу.
2. Скопировать `.env.example` в `.env`.
3. Заполнить `VK_TOKEN`, `VK_TEST_TOKEN`, `DB_*`, `ADMINS`.
4. Для безопасной проверки оставить:

```env
ACTIVE_VK_TOKEN_ENV=VK_TEST_TOKEN
```

5. Установить зависимости и запустить:

```powershell
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe bot.py
```

Бот сам включает VK Long Poll для выбранного сообщества и создает недостающие таблицы.

Если Docker установлен, локальную PostgreSQL можно поднять так:

```powershell
docker compose up -d postgres
```

## Переключение на основное сообщество

После проверки тестового сообщества поменять:

```env
ACTIVE_VK_TOKEN_ENV=VK_TOKEN
```

`VK_GROUP_ID` и `VK_TEST_GROUP_ID` уже вынесены отдельными переменными.

## Важное про VK-площадки

Рекламная площадка в этом боте - это VK-сообщество со стеной. Для добавления
площадки партнер отправляет ссылку/ID сообщества и токен этого сообщества:

```text
https://vk.com/club123456 token=vk1.a.xxxxx
```

Токен нужен с правами `wall`, `photos` и `messages`, чтобы купленные объявления
публиковались на стену выбранных площадок. Подписочный доступ при этом работает
через VK-беседы: для него используются `peer_id` вида `2000001234`, команда `/id`
в беседе пришлет peer_id в личку и удалит команду из чата.

Покупка `Объявление` предлагает клиенту только подходящие VK-сообщества со стеной.
Чаты в этот список не попадают. Цена считается как тариф `poster` за выбранный
период, умноженный на количество выбранных сообществ. Админ меняет базовый тариф
в разделе `Реклама -> Тарифы -> Объявления за сообщество`.

## Оплаты

По умолчанию покупка работает через ручную заявку:

1. пользователь выбирает рекламу и срок;
2. бот считает цену;
3. пользователь отправляет комментарий/скриншот оплаты;
4. админ подтверждает в разделе `Ручные оплаты`;
5. реклама активируется или уходит на модерацию.

Автопроверка YooMoney/YooKassa включается через `MAIN_PAYMENT_TYPE=yoomoney`
или `MAIN_PAYMENT_TYPE=yookassa`. Если платежный провайдер недоступен или
пользователь выбирает ручную проверку, бот возвращается к сценарию ручной заявки.
Для YooMoney можно задать `YOOMONEY_RECEIVER`, иначе используется кошелек из
старой Telegram-версии.

Для YooKassa нужны `YOOKASSA_ACCOUNT_ID` и `YOOKASSA_SECRET_KEY`.
`YOOKASSA_RETURN_URL` можно оставить пустым: бот сам вернет пользователя в диалог
активного VK-сообщества. Платеж создается через redirect-confirmation, с
idempotency key и metadata заказа. Статус проверяется кнопкой `Проверить оплату`;
`succeeded` активирует покупку, `canceled` предлагает ручную проверку или новую
покупку, `waiting_for_capture` автоматически захватывается при
`YOOKASSA_AUTO_CAPTURE_WAITING=true`.
HTTP-запросы к YooKassa выполняются напрямую с короткими таймаутами:
`YOOKASSA_HTTP_TIMEOUT=5`, `YOOKASSA_HTTP_ATTEMPTS=1`,
`PAYMENT_GATEWAY_TIMEOUT=8`. Если API YooKassa недоступен, бот быстро покажет
ручную оплату вместо долгого ожидания.

Для администраторов бота покупки активируются бесплатно: внешний платеж не
создается, сумма записывается как `0`.

## Админка

Доступ задается VK user id:

```env
ADMINS=1113916884
```

Команда `/id` в диалоге с ботом показывает `VK user_id` и `peer_id`.

## Systemd

Шаблон сервиса лежит в `vk_bot.service`. Перед установкой нужно заменить пути под сервер.
